Design — Section 3: Command Details

CA management commands:

Command Flags Description
gossl ca create --cn, --db, --days, --key-algo (rsa|ecdsa) Create root CA, initialize DB
gossl intermediate create --cn, --db, --days, --key-algo Create intermediate signed by root
gossl cert issue --cn, --db, --intermediate, --days, --san, --key-algo Issue cert signed by intermediate
gossl cert revoke --serial, --db Mark cert as revoked
gossl cert export --serial, --db, --out, --chain (include full chain) Export cert PEM to file

Troubleshooting commands:

Command Args/Flags Description
gossl cert inspect <file\|-> Parse and display cert details (subject, issuer, SANs, expiry, fingerprint)
gossl cert verify <file\|->, --chain Verify cert against chain, report errors
gossl csr inspect <file\|-> Parse and display CSR details
gossl server check <host:port>, --sni Connect to server, show cert chain, verify trust, flag issues

TUI:

Command Flags Description
gossl ui --db Launch Bubble Tea TUI for CA management

All DB commands default to --db gossl.db in the current directory if not specified. stdin is indicated by - or absence of a file arg when piped.

Design — Section 6: Error Handling

Errors follow two patterns:

Library errors (internal/) — typed errors that callers can inspect:

// internal/cert/errors.go
var (
    ErrExpired          = errors.New("certificate has expired")
    ErrChainIncomplete  = errors.New("chain of trust is incomplete")
    ErrInvalidCSR       = errors.New("invalid certificate signing request")
)

Wrap with context: fmt.Errorf("verifying %s: %w", cn, ErrChainIncomplete)

CLI errors (cmd/) — translate library errors into human-readable output with exit codes:

Exit code Meaning
0 Success
1 General error (bad flags, file not found)
2 Cert validation failure (expired, bad chain, invalid CSR)
3 DB error (corrupt, missing, schema mismatch)

No panics. All errors returned and handled at the Cobra command level.
