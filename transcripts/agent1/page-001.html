Save Session Cost

Generate cost tracking CSVs from the current session's JSONL logs.

Supports both Oh My Pi (OMP) and Claude Code session formats. Auto-detects based on which session directory exists.

Output

Two files in ~/.claude/cost/YYYY/MM/DD/: - HHMMSS_sessionid_summary.csv — single row with totals - HHMMSS_sessionid_details.csv — per-interaction rows sorted by timestamp

Steps

  1. Detect session source (OMP or Claude Code) and find the current session JSONL
  2. Find subagent JSONLs if present
  3. Run the cost script below
  4. Report total cost and interaction count

Session Discovery

OMP (Oh My Pi)

  • Session dir: ~/.omp/agent/sessions/{PROJECT_DIR}/ (e.g. -dev-cc-main)
  • Session JSONL: most recently modified .jsonl in that directory
  • Session ID: filename stem (e.g. 2026-03-14T22-42-38-656Z_1492ee5870123c42)
  • Subagent dir: {SESSION_DIR}/{SESSION_ID}/ (check for .jsonl files)
  • Detect: first line of JSONL has "type":"session" with "version":3

Claude Code (fallback)

  • Session dir: ~/.claude/projects/{PROJECT_DIR}/ (e.g. -Users-scaruso-dev-cc-main)
  • Session JSONL: most recently modified .jsonl in that directory
  • Session ID: filename stem (UUID)
  • Subagent dir: {SESSION_DIR}/{SESSION_ID}/subagents/

import json, os, csv, glob
from datetime import datetime

PRICING = {
    "claude-opus-4-6": {"input": 5/1e6, "output": 25/1e6, "cache_write": 6.25/1e6, "cache_read": 0.50/1e6},
    "claude-sonnet-4-6": {"input": 3/1e6, "output": 15/1e6, "cache_write": 3.75/1e6, "cache_read": 0.30/1e6},
    "claude-haiku-4-5": {"input": 1/1e6, "output": 5/1e6, "cache_write": 1.25/1e6, "cache_read": 0.10/1e6},
}

def get_pricing(model):
    for key, p in PRICING.items():
        if key in model:
            return p
    return PRICING["claude-opus-4-6"]

def detect_format(filepath):
    """Read first line to detect OMP vs Claude Code format."""
    with open(filepath) as f:
        first = f.readline()
        try:
            entry = json.loads(first)
            if entry.get("type") == "session" and entry.get("version", 0) >= 3:
                return "omp"
        except:
            pass
    return "claude"

def parse_omp_jsonl(filepath, source):
    """Parse OMP session JSONL. Messages use camelCase keys and nested message object."""
    rows = []
    with open(filepath) as f:
        for line in f:
            try:
                entry = json.loads(line)
            except:
                continue
            if entry.get("type") != "message":
                continue
            msg = entry.get("message", {})
            if msg.get("role") != "assistant":
                continue
            stop = msg.get("stopReason", "")
            if not stop or stop == "aborted":
                continue
            usage = msg.get("usage", {})
            model = msg.get("model", "unknown")
            input_tokens = usage.get("input", 0)
            output_tokens = usage.get("output", 0)
            cache_write = usage.get("cacheWrite", 0)
            cache_read = usage.get("cacheRead", 0)
            p = get_pricing(model)
            cost = (input_tokens * p["input"] + output_tokens * p["output"] +
                    cache_write * p["cache_write"] + cache_read * p["cache_read"])
            rows.append({
                "timestamp": entry.get("timestamp", ""),
                "source": source,
                "model": model,
                "provider": msg.get("provider", ""),
                "input_tokens": input_tokens,
                "output_tokens": output_tokens,
                "cache_write_tokens": cache_write,
                "cache_read_tokens": cache_read,
                "cost_usd": round(cost, 6),
                "stop_reason": stop,
                "duration_ms": msg.get("duration", 0),
                "ttft_ms": msg.get("ttft", 0),
            })
    return rows

def parse_claude_jsonl(filepath, source):
    """Parse Claude Code session JSONL. Messages use snake_case keys at top level."""
    rows = []
    with open(filepath) as f:
        for line in f:
            try:
                entry = json.loads(line)
            except:
                continue
            if entry.get("type") != "assistant":
                continue
            msg = entry.get("message", {})
            if not msg.get("stop_reason"):
                continue
            usage = msg.get("usage", {})
            model = msg.get("model", "unknown")
            input_tokens = usage.get("input_tokens", 0)
            output_tokens = usage.get("output_tokens", 0)
            cache_write = usage.get("cache_creation_input_tokens", 0)
            cache_read = usage.get("cache_read_input_tokens", 0)
            p = get_pricing(model)
            cost = (input_tokens * p["input"] + output_tokens * p["output"] +
                    cache_write * p["cache_write"] + cache_read * p["cache_read"])
            rows.append({
                "timestamp": entry.get("timestamp", ""),
                "source": source,
                "model": model,
                "provider": "",
                "input_tokens": input_tokens,
                "output_tokens": output_tokens,
                "cache_write_tokens": cache_write,
                "cache_read_tokens": cache_read,
                "cost_usd": round(cost, 6),
                "stop_reason": msg.get("stop_reason", ""),
                "duration_ms": 0,
                "ttft_ms": 0,
            })
    return rows

def find_subagent_jsonls(session_dir, session_id, fmt):
    """Find subagent JSONL files for a session."""
    if fmt == "omp":
        # OMP: subagent logs are *.jsonl inside {session_dir}/{session_id}/
        sub_dir = os.path.join(session_dir, session_id)
    else:
        # Claude Code: subagent logs are in {session_dir}/{session_id}/subagents/
        sub_dir = os.path.join(session_dir, session_id, "subagents")
    if not os.path.isdir(sub_dir):
        return []
    return [
        (os.path.join(sub_dir, f), f.replace(".jsonl", ""))
        for f in os.listdir(sub_dir)
        if f.endswith(".jsonl")
    ]

def save_cost(session_id, session_dir):
    main_jsonl = os.path.join(session_dir, f"{session_id}.jsonl")
    fmt = detect_format(main_jsonl)
    parse = parse_omp_jsonl if fmt == "omp" else parse_claude_jsonl

    now = datetime.now()
    cost_dir = os.path.expanduser(now.strftime("~/.claude/cost/%Y/%m/%d"))
    os.makedirs(cost_dir, exist_ok=True)

    all_rows = parse(main_jsonl, "main")
    for sub_path, agent_id in find_subagent_jsonls(session_dir, session_id, fmt):
        all_rows.extend(parse(sub_path, agent_id))

    all_rows.sort(key=lambda r: r["timestamp"])

    models = sorted(set(r["model"] for r in all_rows))
    providers = sorted(set(r["provider"] for r in all_rows if r["provider"]))
    total_input = sum(r["input_tokens"] for r in all_rows)
    total_output = sum(r["output_tokens"] for r in all_rows)
    total_cache_write = sum(r["cache_write_tokens"] for r in all_rows)
    total_cache_read = sum(r["cache_read_tokens"] for r in all_rows)
    total_cost = sum(r["cost_usd"] for r in all_rows)
    total_duration = sum(r["duration_ms"] for r in all_rows)

    ts = now.strftime("%H%M%S")

    summary_path = os.path.join(cost_dir, f"{ts}_{session_id}_summary.csv")
    with open(summary_path, "w", newline="") as f:
        w = csv.writer(f)
        w.writerow(["session_id", "format", "providers", "models",
                     "total_input_tokens", "total_output_tokens",
                     "total_cache_write_tokens", "total_cache_read_tokens",
                     "total_cost_usd", "num_interactions", "total_duration_ms"])
        w.writerow([session_id, fmt, ";".join(providers), ";".join(models),
                     total_input, total_output, total_cache_write, total_cache_read,
                     round(total_cost, 4), len(all_rows), total_duration])

    details_path = os.path.join(cost_dir, f"{ts}_{session_id}_details.csv")
    fields = ["timestamp", "source", "model", "provider",
              "input_tokens", "output_tokens", "cache_write_tokens", "cache_read_tokens",
              "cost_usd", "stop_reason", "duration_ms", "ttft_ms"]
    with open(details_path, "w", newline="") as f:
        w = csv.DictWriter(f, fieldnames=fields)
        w.writeheader()
        w.writerows(all_rows)

    main_count = len([r for r in all_rows if r["source"] == "main"])
    sub_count = len(all_rows) - main_count
    print(f"Format: {fmt}")
    if providers:
        print(f"Provider: {', '.join(providers)}")
    print(f"Interactions: {len(all_rows)} ({main_count} main, {sub_count} subagent)")
    print(f"Tokens: {total_input:,} in / {total_output:,} out / {total_cache_write:,} cache-w / {total_cache_read:,} cache-r")
    print(f"Total cost: ${round(total_cost, 4)}")
    if total_duration > 0:
        print(f"Total API time: {total_duration / 1000:.1f}s")
    print(f"Summary: {summary_path}")
    print(f"Details: {details_path}")

- **Effective Go**: https://go.dev/doc/effective_go
- **Go Code Review Comments**: https://go.dev/wiki/CodeReviewComments

## Key Rules from Effective Go

### Formatting
- Use `gofmt` — no exceptions. All formatting debates are settled by the tool.
- Use tabs for indentation, spaces for alignment.

### Naming
- Use MixedCaps/mixedCaps, never underscores for multi-word names.
- Package names: short, concise, lowercase, single-word. No underscores or mixedCaps.
- Getters: use `Owner()` not `GetOwner()`. Setters: `SetOwner()` is fine.
- Single-method interfaces: name by method + "er" suffix (e.g., `Reader`, `Writer`, `Formatter`).
- Exported names are qualified by package — don't stutter (e.g., `ring.Buffer` not `ring.RingBuffer`).

### Control Flow
- Prefer early returns / guard clauses — avoid deep nesting.
- `if` init statements: use to set up local variables (`if err := doThing(); err != nil`).
- No unnecessary `else` — if `if` ends in `break`/`continue`/`return`, drop the `else`.
- Use `switch` over long `if-else` chains.

### Error Handling
- Always check errors. Don't discard them with `_`.
- Error strings: lowercase, no punctuation — they're often composed into larger messages.
- Use `fmt.Errorf` with `%w` for wrapping errors with context.
- Handle errors at the appropriate level. Don't just log and return — do one or the other.

### Functions & Methods
- Use named return values for documentation when the meaning isn't obvious.
- Prefer pointer receivers for consistency within a type, and when the method needs to mutate the receiver.
- Keep functions short and focused. If a function is long, consider splitting it.

### Concurrency
- Share by communicating — prefer channels over shared memory with mutexes.
- Use goroutines only when the work is actually concurrent.
- Be explicit about goroutine lifetimes — always know how a goroutine will exit.
- Use `sync.WaitGroup` or `context.Context` for coordination.

### Interfaces
- Define interfaces where they are used, not where they are implemented.
- Keep interfaces small — one or two methods.
- Accept interfaces, return concrete types.

### Data
- Use `make` for slices, maps, channels. Use `new` or `&T{}` for structs.
- Prefer slices over arrays.
- Use `init()` sparingly — it makes code harder to test.

## Key Rules from Code Review Comments

### Documentation
- All exported names must have doc comments.
- Doc comments are complete sentences starting with the name being declared.
- Package comments: one file should have a `// Package foo ...` comment.

### Contexts
- Functions that do I/O or are long-running should accept `context.Context` as the first parameter.
- Don't store contexts in structs. Pass them as function parameters.

### Copying
- Don't copy values with embedded locks (`sync.Mutex`). Pass by pointer.

### Crypto Rand
- Use `crypto/rand` for generating keys/secrets, never `math/rand`.

### Declaring Empty Slices
- Prefer `var s []T` (nil slice) over `s := []T{}` (empty, non-nil) unless JSON encoding requires non-null.

### Don't Panic
- Don't use `panic` for normal error handling. Return errors instead.
- Panics are for truly unrecoverable situations in program initialization.

### Error Strings
- Lowercase, no trailing punctuation: `fmt.Errorf("something bad")` not `fmt.Errorf("Something bad.")`.

### Goroutine Lifetimes
- Document goroutine lifetimes — when and why they terminate.
- Avoid leaking goroutines — always provide a way to stop them.

### Handle Errors
- Don't discard errors. Don't use `_ = f()` when `f` returns an error.
- If truly ignorable, document why.

### Imports
- Group imports: stdlib, then a blank line, then everything else.
- Use `goimports` to manage import ordering.
- Avoid renaming imports except to avoid conflicts.
- Never use `.` imports except in test files that use the testing pattern.

### In-Band Errors
- Don't overload return values. Use `(value, ok)` or `(value, error)` patterns.

### Indent Error Flow
- Keep the "happy path" at minimal indentation. Handle errors first, return early.

### Initialisms
- Keep initialisms consistent casing: `URL`, `HTTP`, `ID` — not `Url`, `Http`, `Id`.
- In unexported names: `urlClient`, `httpServer`, `userID`.

### Line Length
- No fixed limit but avoid unreasonably long lines. Break for readability.

### Named Result Parameters
- Use them for documentation in short functions. Avoid bare `return` statements.

### Package Comments
- Every package needs a package comment. For multi-file packages, only one file needs it.

### Receiver Names
- Short (1-2 letters), consistent across all methods, never `this` or `self`.

### Receiver Type
- Use pointer receivers when the method modifies the receiver, the receiver is large, or for consistency.
- Use value receivers for small immutable types and basic types.

### Synchronous Functions
- Prefer synchronous functions over async ones. Let the caller manage concurrency.

### Test Failures
- Use meaningful test failure messages: include what input was, what was expected, what was received.
- Use `t.Errorf("Foo(%q) = %d, want %d", input, got, want)` pattern.

### Variable Names
- Short names for short scopes: `i`, `r`, `buf`.
- Longer, more descriptive names for larger scopes.
- Avoid redundancy with context: `count` not `sliceCount` when it's already clear.

**How to apply:** Use these rules during all code review (both human and AI). When generating Go code, follow these conventions by default. Reference specific rules when suggesting changes in review.

1. Effective Go -- Key Guidelines

Formatting

  • Always run gofmt (or goimports). All mechanical style debates are settled by the tool.
  • Use tabs for indentation, spaces only when necessary.
  • No strict line length limit; if a line feels too long, wrap and indent with an extra tab.

Naming

  • Package names: short, lowercase, single-word. No underscores or mixedCaps. Avoid meaningless names like util, common, misc.
  • Avoid stutter: don't repeat the package name in exported identifiers. Use ring.New(), not ring.NewRing().
  • Getters: name them Owner(), not GetOwner(). Setters are SetOwner().
  • Interface names: one-method interfaces get an -er suffix (Reader, Writer, Stringer).
  • MixedCaps: always use MixedCaps or mixedCaps, never underscores (maxLength, not MAX_LENGTH).

Control Flow

  • Omit unnecessary else: if the if body ends in return, break, continue, or goto, drop the else and keep the happy path unindented.
  • Opening brace on same line as if/for/switch (required by semicolon insertion rules).

Functions and Methods

  • Use multiple return values instead of in-band error signals (-1, null, etc.).
  • defer for resource cleanup (close files, unlock mutexes) -- place it right after acquisition.
  • Pointer vs. value receivers: pointer methods can modify the receiver; value methods cannot. Value methods can be called on pointers and values, but pointer methods only on pointers.

Data Structures

  • new(T) returns a zeroed *T; make is for slices, maps, and channels only and returns an initialized (not zeroed) value.
  • Design for useful zero values: a sync.Mutex or bytes.Buffer should work without explicit initialization.
  • Use composite literals with field names: &File{fd: fd, name: name}.

Interfaces

  • Interfaces are satisfied implicitly -- no implements keyword. Keep interfaces small (one or two methods).
  • If a type exists only to implement an interface, don't export the type; export only the interface.

Concurrency

  • Don't communicate by sharing memory; share memory by communicating (channels).
  • Goroutines are cheap but you must still track their lifetimes.
  • Use channels for synchronization and orchestration; use sync.Mutex only for protecting internal state.

Error Handling

  • Don't panic for normal errors. Return error values and check them.
  • Use panic/recover only for truly unrecoverable situations (like programmer bugs caught in init).

Formatting and Imports

  • Indent error flow: keep the happy path at minimal indentation. Handle errors first, then return/continue, then proceed with normal logic.
  • Don't panic for normal error handling.

# gossl

Go project using Bazel for build and test.

## Build System

- **Bazel** is the build system. Use `bazel build //...` and `bazel test //...`.
- Do not use `go build` or `go test` directly — always use Bazel.
- Keep `BUILD.bazel` files up to date when adding/removing Go files.
- Use `gazelle` to regenerate BUILD files: `bazel run //:gazelle`.

## Go Style

Follow Google's official Go guidelines:

- **Effective Go**: https://go.dev/doc/effective_go
- **Code Review Comments**: https://go.dev/wiki/CodeReviewComments

Key rules enforced in this project:

- Run `gofmt` / `goimports` on all code.
- MixedCaps naming, no underscores. Initialisms in consistent case (`URL`, `ID`, not `Url`, `Id`).
- Package names: short, lowercase, single-word. No stutter (`ring.Buffer` not `ring.RingBuffer`).
- Error strings: lowercase, no trailing punctuation.
- Always check errors. Never discard with `_` unless documented why.
- Early returns for error handling — keep happy path at minimal indentation.
- Interfaces: small, defined at consumer, not producer. Don't predefine for mocking.
- `context.Context` as first parameter. Never store in structs.
- Pointer receivers for mutation/large structs/consistency. Never mix receiver types on a type.
- Receiver names: 1-2 letters, consistent, never `this`/`self`.
- Prefer synchronous APIs. Let callers manage concurrency.
- Document goroutine lifetimes. Don't leak goroutines.
- Table-driven tests. Failure messages: `Foo(%q) = %d, want %d`.
- `crypto/rand` for secrets, never `math/rand`.
- `var s []T` for nil slices, not `s := []T{}` (unless JSON requires `[]`).
- All exported names have doc comments — complete sentences starting with the name.

## Project Structure

```
gossl/
  cmd/           # main packages
  pkg/           # library packages
  internal/      # private packages
  BUILD.bazel    # root build file
  MODULE.bazel   # bazel module definition
  WORKSPACE      # bazel workspace (if not using bzlmod)
  go.mod         # go module file
```
