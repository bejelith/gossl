To support it we need:

  1. Schema change: intermediate table gets a nullable parent_intermediate_id (self-referencing FK) so intermediates can have parent intermediates, not just the root CA
  2. CLI change: intermediate create gets an optional --parent flag to specify the parent intermediate (defaults to root CA if omitted)
  3. Signing change: when --parent is set, sign with the parent intermediate's key instead of the root CA's key
